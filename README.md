Quiz Leaderboard System - Backend Integration Task1. Project OverviewThis project is a Java-based backend application designed to interface with a quiz show validator API. The primary objective is to compute the total score of all participants combined and generate a deduplicated leaderboard by processing 10 separate API polls.The core challenge involves handling idempotency in a distributed system, where the same API response data may be delivered across multiple calls.2. Technical RequirementsTo successfully complete the assignment, the following constraints were met:Polling: The application executes 10 GET requests to the validator.Rate Limiting: A mandatory 5-second delay is maintained between each poll.Deduplication: Data is deduplicated using a composite key: roundId + participant.Aggregation: Scores are aggregated per participant only for unique events.Sorting: The final leaderboard is sorted by totalScore in descending order.Idempotent Submission: The final result is submitted exactly once via a POST request.3. Implementation DetailsHandling Duplicate DataIn this system, processing duplicates would result in an incorrect final score.Incorrect Logic: Adding every score received (e.g., Alice +10 in Poll 1 and Alice +10 again in Poll 3 = 20) .Correct Logic: Using a tracking mechanism (like a HashSet) to ensure that if a specific round for a participant has already been processed, it is ignored in subsequent polls .Expected API FlowPoll: GET /quiz/messages?regNo={regNo}&poll={0-9} .Deduplicate: Filter the events array.Submit: POST /quiz/submit with the final JSON payload .4. How to RunEnsure you have Java 11+ installed.Add the org.json library to your project dependencies.Replace the REG_NO constant in the source code with your registration number.Run the main method.Monitor the console for polling status and the final validation response from the server.5.

##Quiz Leaderboard System - Backend IntegrationProject OverviewThis repository contains the solution for the SRM Internship Java Qualifier. The application simulates a real-world backend problem where it must consume API responses from an external validator, handle data consistency (idempotency), and generate a final deduplicated leaderboard.Technical Approach1. Data Collection & PollingThe application polls the validator API exactly 10 times (index 0-9).A mandatory 5-second delay is maintained between each request to comply with system requirements.2. Deduplication Logic (Handling Idempotency)In distributed systems, duplicate data delivery is common. To ensure accuracy:A unique composite key is generated for every event: roundId + participant.The application uses a Set to track these keys; if a key is encountered again in a later poll, the data is ignored.This prevents the "Double Counting" error where a single round's score is added multiple times.3. Aggregation & SortingUnique scores are aggregated per participant into a Map<String, Integer>.The final list is sorted by totalScore in descending order to generate the leaderboard.A final grand total is computed across all users to verify the submission.API DetailsGET Endpoint: /quiz/messages (Used for data polling).POST Endpoint: /quiz/submit (Used for the final one-time submission).How to RunClone the repository.Update the regNo variable in the code with your specific registration number.Run the main class.The console will log the polling progress and the final API response.
